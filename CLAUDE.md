# Git

Nigdy nie commituj ani nie pushuj zmian do gita, chyba że użytkownik jawnie o to poprosi.

# Backlog

Gdy tworzony jest plan implementacji, sprawdź BACKLOG.md i zaktualizuj pasujący wpis — dodaj link do planu (i speca jeśli istnieje).

Jeśli wykryjesz, że jakiś task z BACKLOG.md został dowieziony (np. feature działa, plan wykonany, kod wdrożony), przenieś go do BACKLOG_DONE.md — pliku z ukończonymi zadaniami. Jeśli BACKLOG_DONE.md nie istnieje, utwórz go. Podczas przenoszenia usuń task kompletnie z BACKLOG.md — zarówno wiersz z tabeli priorytetów, jak i sekcję z opisem (jeśli istnieje).
